SDET / Test Automation Developer

I am a passionate test automation developer, currently working as an SDET with a product based company in Bay Area. My daily job is to write test scripts to automate the REST APIs and the front-end using the technology stack such as nodejs, mochajs, jasmine, chai, protractor and test-cafe with javascript as language binding.I am well adept in developing the automation framework from scratch based on the requirements. I have a strong inclination towards open-source tools and continue to strive towards make it tailor-made as per project circumstances.

I am more passionate towards DevOps methodology using Jenkins, Github, Docker and another continuous testing concept. I particularly enjoy working in Test Driven Development and Business Driven Development approach.

I have a goal to significantly contribute to the quality of software products and to leave the greatest fingerprint on people’s mind by making the world better place.

Here are few Key-Points about me:

  • Understanding requirements, technical specifications and writing Manual & Automation test cases based on business requirements.
  • Experienced in developing test automation frameworks based on the requirement.
  • Implemented UI Automation using Selenium WebDriver, JAVA, Selenium Grid, Cucumber-JVM, Maven, and Protractor.
  • Implemented RESTful web services automation using Java, Maven, Selenium Webdriver, TestNG, Cucumber, GSON, Rest-assured, Mocha and Chai.
  • Developed BDD tests using Cucumber by writing behaviors and step definitions. Developed required Selenium support code in JAVA for Cucumber.
  • Worked in a highly dynamic AGILE environment and participated in scrum and sprint meetings.
  • Performed Defect Tracking & Management in JIRA.
  • Possess out of the box thinking, problem-solving and decision-making skills to improve processes, systems, and methodologies.
  • Positive attitude towards any kind of challenging work.
  • Flexible to learn new technologies.
